Formal Verification

Verification is a research area based on the foundation of proving the validity and truth of any given data. The results of rearch in this area underlies almost all modern technologies, as it can check the accuracy of data that has either been entered or received.

But that's just the beginning...

Main Research Topics

There are 2 main research topics under this area.

Boosted automatic verification develops techniques to detect bugs and errors in software by automating the testing phase.
This can be done in ways:
Static verification checks the code for bugs before it runs.
 Dynamic verification finds and fixes bugs after it has run.

Program verification develops methods of analysing programs to ensure that they are understandable by other engineers. The aim is to produce code tailored to the specific needs of the developer.
The results of this topic have widespread uses, for in the modern age programs developed in one field are often used by engineers in other disciplines. This process is most evident in the electrical and mechanical industries.

Key contributions and Main Achievements


Program verification algorithms are actually used on most modern operating systems. The repair utilities on an operating system automatically find the locations of bugs which a repair tool can then repair without the need of the developer.


Verification-related findings also play a vital role in the gaming market. The code for the Graphical Processing Unit (GPU), responsible for all graphical output tasks, is actually developed using automatic verification techniques.

The low level code required for the GPU to function is its kernel code - previously this was developed by humans, however nowdays it is developed using automatic verification algorithms. The algorithms implement optimal memory usage and identify bugs faster than any human could.

As a result of these increases in efficiency, the UK has experienced economic growth in it's gaming industry as there is currently high demand for GPUs.


UCL's PPLV Research department is currently studying the use of boosted auto-verification techniques in the development of android phone applications. There are over 250,000,000 of them in use so using such as a technique will significantly increase the speed of finding crashes, bugs and memory problems.

Final Thoughts

Formal Verification is a Research Area that underpins most of the modern world's technologies - as such it's a good thing that we have the best teams in the world working on it.
The UCL PPLV Research Group is leading the development of these amazing new technologies.
